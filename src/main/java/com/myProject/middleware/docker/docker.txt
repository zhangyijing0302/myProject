docker思想来源于集装箱
打包装箱，每个箱子之间隔离
docker通过隔离机制，将服务器性能用到极致

 虚拟机缺点：
    1.资源占用多
    2.冗余步骤多
    3.启动很慢
    4.传统虚拟机，虚拟出一套硬件，运行一个完整的操作系统，在该操作系统上安装运行软件
 容器化技术：
    1. 容器不是完整的模拟操作系统
    2. 应用在宿主机上，容器没有自己的内核，也没有虚拟出硬件，所以是轻量级的，每个容器之间互相隔离。每个容器都有一个自己的文件系统


镜像image:
    模板，可以通过模板创建容器服务，mysql镜像 -》 run -》 mysql容器01
容器container
    独立运行一个或一个组的应用，通过镜像来创建
    通过基本命令操作
仓库repository
    存放镜像的地方
    有公共仓库和私有仓库
    Docker Hub
    通过配置阿里云镜像加速

docker运行镜像
    docker会在本机寻找镜像 --》 本机是否有镜像 --》存在 --》运行
                                         --》不存在 docker Hub下载 --》 可以下载 --》 运行
                                                                 --》 不可以下载 --》 报错

docker是怎么工作的
    docker是CS架构，Docker的守护进程运行在主机，通过socket从客户端访问
    docker-Server接收到Docker-Client的指令，就会执行这个命令。

docker为什么比VM快？
    Docker有更小的抽象层
    运行两个centos虚拟机
    运行一个系统，通过docker去运行两个服务，两个容器
    docker利用的是宿主机的内核，新建一个容器不需要重新加载内核（秒级），
    vm需要重新搭建一套操作系统，重新启动内核（分钟级别）
